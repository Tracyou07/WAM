Academic Report · Embodied AI · World Models

World Action Models：从世界建模到动作生成

本汇报的核心观点是：WAM 不是“视频生成模型加一个动作头”，而是一类试图把 世界动态表征、动作分布与规划价值 放入同一序列决策问题中的模型范式。

Central Thesis

World Model 解决“世界如何演化”；WAM 进一步追问“为了完成任务，动作与未来世界应如何联合建模”。

01

Problem Formulation

为什么不能只学 observation 到 action

在具身控制中，智能体面对的是部分可观测环境。视觉观测只能给出当前场景的投影， 但动作的好坏取决于它如何改变未来世界。直接策略模型可以学习从图像和语言到动作的映射， 但它未必显式约束模型理解接触、遮挡、物体运动和任务进展。

因此，WAM 的学习路线应从序列决策出发：当前观测不是终点，而是推断隐藏世界状态的证据； 动作不是孤立输出，而是会改变未来观测分布的干预变量。

Policy / VLA p(action | observation, language) World Model p(future observation | past observation, action) WAM p(future observation, future action | past observation, language)
02

Conceptual Framework

World Model 与 WAM 的概念边界

World Model

World Model 的主要对象是环境动力学。它把动作作为条件，预测世界随后如何变化。 在传统控制中，这对应运动方程、观测模型和状态估计；在神经模型中，这些结构被压缩进 latent representation。

World Action Model

WAM 将动作从“给定条件”提升为“建模对象”。它不只问某个动作会导致什么未来， 还问在任务条件下哪些动作与哪些未来状态共同构成高概率或高价值轨迹。

模型 核心问题 动作的角色 典型用途
World Model 动作之后世界如何变化 条件变量 预测、仿真、rollout
Policy / VLA 当前应该执行什么动作 输出变量 直接控制
WAM 动作与未来世界如何联合产生 条件相关且可生成 控制、规划、表征学习
03

Model-Based Reasoning

Model、Planning 与 Synthetic Rollout

学到的 model 是一个可查询的近似环境；synthetic rollout 是在这个近似环境中展开出的虚拟未来； planning 则是用这些虚拟未来比较候选动作。三者的关系不是并列，而是逐层依赖。

1

Learned Model

从真实数据中学习动作条件下的未来预测。

2

Synthetic Rollout

在模型内部生成多条可能未来轨迹。

3

Planning

根据回报或 value 对候选未来排序，并执行最优动作。

这里的 planning 与 Bellman 思想一致：动作价值由当前收益和后继状态的未来价值共同决定。 但 WAM 是否真正“使用 Bellman”，取决于它是否显式建模 reward、return、value 或 Q function。

04

Three Paradigms

三种 WAM 范式的核心差异

Paradigm A

Joint Video-Action Modeling

将未来视觉 token 与动作 token 作为同一个生成对象。优点是动作和未来相互约束； 代价是推理通常需要生成高维未来内容，计算开销较高。

Paradigm B

Imagine-Then-Execute

先生成目标未来，再由动作模型或 inverse dynamics 反推出动作。结构清晰， 但生成未来和可执行动作之间可能存在分布错配。

Paradigm C

Fast-WAM

训练时保留视频预测目标，推理时不显式生成未来视频。它将 future prediction 从测试时规划步骤转化为训练时的表征学习约束。

关键分歧： WAM 的收益究竟来自测试时显式 imagination，还是来自训练时视频预测所塑造的 latent world representation？
05

Diffusion Objective

扩散模型在这里不是“画图工具”

在 WAM 语境中，扩散模型的生成对象可以不是图像本身，而是未来视觉 latent、动作 token、 proprioception token 甚至 value token 的联合表示。训练时，模型学习从带噪的联合样本中恢复真实演示轨迹。

干净样本
未来世界表示与动作表示的拼接，例如 future visual latent + action token。
条件信息
过去观测、语言指令、机器人状态等上下文。
训练任务
预测加入到样本中的噪声，从而学会条件去噪。
控制意义
不是为了生成漂亮视频，而是为了学习多峰、动作相关、物理一致的未来分布。

Fast-WAM 的重要性在于：它保留扩散/视频预测带来的表征学习收益，但在推理阶段不再支付显式未来生成的延迟成本。
